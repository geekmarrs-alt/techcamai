Developer Preview · Honest breakdown

What works. What doesn't.

This is a pre-production Developer Preview. Features listed as Built are working in the repo today. Planned items are on the roadmap, not shipped.

Built Working in the repo now Beta Works but rough edges Planned Not yet built
⚠️ Developer Preview — not production-ready
  • No authentication: the operator console has no login. Anyone on the same network can access it.
  • Credentials are stored in plaintext in the SQLite database on the Pi. This is an MVP trade-off, not a final design.
  • No multi-tenancy. No cloud hosting. No billing infrastructure built yet.
  • Clip capture can fail on cameras with slow RTSP response times — status tracked in the alert inbox.
  • The detection model is configurable but the default integration is the internal worker — no external AI provider support yet.
📡
Camera management
Discover, add, and manage cameras on your LAN
LAN scanning Built

Scans your local network, tries common snapshot endpoints, and shows vendor hints to help identify cameras quickly.

Digest auth support Built

Works with cameras requiring HTTP Digest authentication (common on Hikvision, Dahua, Reolink). Stores credentials per-camera.

httpx async · per-camera credential store
Snapshot preview on add Built

When adding a camera, fetch a live snapshot immediately to confirm credentials and URL are correct before saving.

Per-camera enable/disable Built

Toggle individual cameras in or out of the monitoring loop without deleting their configuration.

Inline editing Built

Edit camera name, IP, channel, and credentials directly in the Cameras management page without leaving the console.

ONVIF auto-discovery Planned

Proper ONVIF device discovery and stream URI resolution. Currently uses heuristic URL probing.

🔴
Live monitoring
Real-time snapshot wall and alert polling
Live snapshot wall Built

Multi-camera grid that auto-refreshes snapshots on a configurable interval (2–10 seconds). One-click access from the sidebar.

Server-side snapshot proxy · 2–10s refresh
Alert polling Built

Console polls for new alerts every 2.5 seconds. Page reloads automatically when new detections arrive so the inbox stays current.

Vanilla JS IIFE · /api/alerts/latest endpoint
Live RTSP stream Planned

Full RTSP video stream in-browser via HLS or WebRTC transcoding. Currently snapshot-only (JPEG polling).

WebSocket push Planned

Replace the polling approach with WebSocket push for true real-time alert delivery without page reloads.

🔔
Alerts & triage
Detection events, thumbnails, and operator workflow
Alert inbox Built

Full detection history with snapshot thumbnail, camera name, rule name, confidence score, and timestamp. Filter by unacknowledged.

One-click acknowledge Built

Mark alerts as reviewed to clear the inbox. Acked state persisted in the database.

Event timeline Built

24-hour hourly activity strip plus a scrollable chronological event list. Useful for post-incident review without trawling the full inbox.

Confidence threshold rules Beta

Per-camera rules with configurable label, confidence threshold, and cooldown period. Reduces false positives from high-sensitivity cameras.

label · conf_threshold · cooldown_seconds
Email / webhook notifications Planned

Push alerts to email or an external webhook URL (Slack, Teams, custom integrations). Not yet built.

Alert search & export Planned

Search by camera, label, or date range. Export to CSV for reporting or evidence packs.

🎬
Clip capture & playback
Post-trigger video recording via RTSP
Post-trigger clip capture Beta

On detection, the worker captures a ~12-second MP4 clip from the RTSP stream. Clip status tracked: pending → ready | failed.

ffmpeg · RTSP · MP4 · ~12s duration
In-console playback Beta

Clip plays directly in the alert inbox when ready. No separate player or download required. Video served via /clips/ endpoint.

HTML5 video · /clips/{path} stream
Pre-trigger buffer Planned

Ring buffer to capture footage before the detection event, not just after. Requires persistent RTSP stream ingestion.

Retention management Planned

Auto-delete old clips based on age or disk usage threshold. Disk health shown in the console.

🥧
Deployment & infrastructure
Edge-first, Pi-native, Docker-based
Raspberry Pi install Built

One-liner install script. Docker installed automatically if missing. Compose stack starts API, worker, and Watchtower.

Pi OS Lite/Desktop · arm64/amd64 · Docker Compose
Watchtower auto-updates Built

Watchtower watches the GHCR image registry and applies updates automatically. Push to master, Pi pulls the new :stable image.

GHCR · :stable tag · Watchtower
Multi-arch images Built

GitHub Actions builds and pushes arm64 + amd64 images to GHCR on every master commit.

GitHub Actions · GHCR · arm64 + amd64
SQLite local DB Built

No external database required. SQLite on a shared Docker volume. Works offline. Simple to back up.

SQLite3 · SQLModel · /data volume
Authentication Planned

Login screen for the operator console. Currently open — anyone on the local network can access it.

Multi-site dashboard Planned

Aggregate alerts and camera status across multiple Pi deployments in a single hosted view.
